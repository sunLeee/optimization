# 보안 핵심 원칙

| 원칙 | 필수 |
|------|------|
| **민감정보** | 하드코딩 금지, 환경변수 사용 |

**상세**: API 키, 비밀번호, 토큰은 환경변수 또는 시크릿 매니저 사용. `.env`는 `.gitignore`에 포함.

| **입력 검증** | 모든 사용자 입력 검증 (Pydantic) |

**상세**: Pydantic 모델로 타입/길이/패턴 검증. XSS 방지를 위한 이스케이프. 화이트리스트 기반 검증 권장.

| **SQL Injection** | ORM 사용, raw SQL 금지 |

**상세**: SQLAlchemy ORM 쿼리 사용. 불가피한 raw SQL은 `text()` + 파라미터 바인딩 필수.

| **로깅** | 비밀번호, 토큰 로깅 절대 금지 |

**상세**: 로그 출력 전 민감정보 마스킹. 사용자 식별 정보도 익명화/해시화 고려. 감사 로그 분리.

| **의존성** | 알려진 취약점 패키지 사용 금지 |

**상세**: `pip-audit`, `safety` 정기 실행. Dependabot 활성화. 취약점 발견 시 즉시 업데이트.

**상세 가이드**: [@skills/convention-security-hardening/SKILL.md] 참조
