# DB 마이그레이션 핵심 원칙

| 원칙 | 필수 |
|------|------|
| **명명 규칙** | `{timestamp}_{description}.py` 형식 |

**상세**: `20240115_add_user_email.py` 형태. Alembic은 자동 생성. 설명은 영문 snake_case.

| **원자성** | 하나의 마이그레이션 = 하나의 논리적 변경 |

**상세**: 테이블 추가와 컬럼 추가를 분리. 실패 시 부분 적용 방지. 트랜잭션 단위 관리.

| **롤백 가능** | downgrade() 함수 필수 구현 |

**상세**: 모든 `upgrade()` 변경에 대응하는 `downgrade()` 작성. 역방향 마이그레이션 테스트 필수.

| **데이터 보존** | DROP 전 데이터 백업 로직 포함 |

**상세**: 컬럼/테이블 삭제 시 데이터 이관 또는 백업 스크립트 선행. `op.execute()`로 백업 쿼리 실행.

| **테스트** | 마이그레이션 전후 테스트 필수 |

**상세**: 빈 DB에서 전체 마이그레이션 테스트. 프로덕션 스키마 덤프와 비교 검증.
