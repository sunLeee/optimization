# ADR-039: 서브에이전트 컨텍스트 25% 이하 유지 규칙

- **상태**: Accepted
- **날짜**: 2026-03-19
- **범용 여부**: ✅ 모든 프로젝트 적용 가능

## 맥락
서브에이전트에 전체 히스토리/대형 문서를 통째로 넘기면 컨텍스트 창이 포화되어 에이전트 성능 저하 및 비용 증가.

## 결정
서브에이전트에는 현재 Task에 필요한 **최소 컨텍스트**만 전달한다. 컨텍스트 창의 **25% 이하** 유지 목표.

## 이유
- 200K 모델 → 50K, 1M 모델 → 250K 이하 유지
- 컨텍스트 포화 시 에이전트가 앞부분을 잊어버림 (lost-in-the-middle)
- 불필요한 히스토리 전달 = 토큰 낭비

## 대안 검토

| 대안 | 거부 이유 |
|------|---------|
| 전체 히스토리 전달 | 컨텍스트 포화, 비용 증가 |
| 요약 없이 발췌 | 컨텍스트 단절 위험 |

## 결과
- 요약본 + 핵심 발췌로 구성하여 전달
- 이미 읽은 파일은 요약을 함께 제공
- 완료된 Task, 관련 없는 항목, 대규모 카탈로그 제외
