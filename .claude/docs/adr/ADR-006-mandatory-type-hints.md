# ADR-006: Python 타입 힌트 필수 적용

- **상태**: Accepted
- **날짜**: 2026-03-19

## 맥락
타입 힌트 없는 Python 코드는 IDE 자동완성, 버그 조기 탐지, 문서화가 부족함.

## 결정
모든 public 함수/메서드의 파라미터와 반환값에 타입 힌트 **필수**. `mypy --strict` 통과 기준.

## 이유
- 런타임 전 타입 오류 탐지
- IDE 지원 향상 (자동완성, 리팩토링)
- 코드 자체가 문서 역할
- AI agent가 함수 시그니처만으로 사용법 파악 가능

## 대안 검토
| 대안 | 거부 이유 |
|------|---------|
| 선택적 타입 힌트 | 일관성 없음, mypy 효과 반감 |
| docstring으로 대체 | 타입 검사 불가 |

## 결과
- `mypy --strict` pre-commit 강제
- `Optional` 대신 `X | None` 문법 (Python 3.10+)
