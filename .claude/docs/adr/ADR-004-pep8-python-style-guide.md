# ADR-004: PEP8을 Python 코딩 스타일 가이드로 채택

- **상태**: Accepted
- **날짜**: 2026-03-19

## 맥락
Python 코드 스타일 표준이 없으면 팀원마다 다른 형식으로 작성하여 코드 리뷰 시 스타일 논쟁 발생.

## 결정
**PEP8**을 Python 코드의 공식 스타일 가이드로 채택. ruff로 자동 강제.

## 이유
- Python 공식 표준. 업계 가장 넓은 지지.
- ruff가 PEP8 체크 자동화 → 사람이 스타일 논쟁하지 않아도 됨
- IDE 플러그인 광범위 지원

## 대안 검토
| 대안 | 거부 이유 |
|------|---------|
| Google Style | PEP8 상위 호환이지만 일부 규칙 충돌 |
| 스타일 가이드 없음 | 코드 리뷰 비용 증가 |

## 결과
- 모든 .py 파일은 ruff 통과해야 함 (pre-commit 강제)
- 79자 줄 길이 (ADR-005 참조)
