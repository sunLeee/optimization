# ADR-016: 계획 수립에 PRD 문서 대신 ralplan 사용

- **상태**: Accepted
- **날짜**: 2026-03-19

## 맥락
작업 시작 전 계획을 문서(PRD)로 작성 후 진행하는 방식과 ralplan으로 동적 합의하는 방식 비교 필요.

## 결정
ralplan(Planner → Architect → Critic 합의 루프)을 주된 계획 도구로 사용.

## 이유
- **즉시성**: PRD 작성 시간 없이 실시간 합의
- **다관점**: Planner/Architect/Critic 3단계 검증 → 단일 작성자 bias 제거
- **측정 가능**: ralplan 종료조건 10개 이상 정량적 기준 (AW-008)
- **적응성**: 요구사항 변화에 즉시 재계획 가능

## 대안 검토
| 대안 | 거부 이유 |
|------|---------|
| PRD 문서 먼저 | 작성 비용 + 문서가 실제 구현과 괴리 |
| 즉시 구현 | 모호한 요구사항으로 재작업 증가 |

## 결과
- ralplan이 PRD 역할 수행
- prd.json으로 story 추적은 ralph 내부에서 처리
